Crash dump is being written to: /usr/local/var/log/rabbitmq/erl_crash.dump...done

原因是1883端口被占用了，使用lsof -i:1883 可找到占用该端口的进程，然后杀死该进程即可，例如：

lsof -i:1883
COMMAND  PID  USER   FD   TYPE             DEVICE SIZE/OFF NODE NAME
java    9705 xiekc  138u  IPv6 0xb5b23c47a591fc29      0t0  TCP *:ibm-mqisdp (LISTEN)

杀死PID为 9705的进程，再执行rabbitmq-server 即可正常启动rabbitmq

rabbitmq-server
